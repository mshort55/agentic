# Agentic Workflow System

This project contains a multi-agent system for analyzing design specs and generating analysis reports using specialized domain expert agents.

## Two Systems, Two Phases

This project uses two complementary systems:

### Phase 1: Analysis (Domain Expert Agents)
**When:** You need to understand *what* to build and *how* to build it well.

Use `/analyze-spec <spec>.md` to consult domain expert agents (Go, Kubernetes, controllers, CRDs, unit testing, e2e testing, coding practices) in parallel. They analyze the design spec and produce a synthesized analysis report with best practices, risks, and recommendations. The report is saved to `output/analysis-reports/`.

- Orchestrator: `.claude/agents/orchestrator/design-spec-orchestrator.md`
- Domain experts: `.claude/agents/domain-experts/*.md`
- Agent config: `config/agents.yaml`

### Phase 2: Execution (Superpowers)
**When:** You have an analysis report and need to *execute* it systematically.

**Requires:** superpowers plugin — install with `/plugin install superpowers@claude-plugins-official`

Use superpowers skills for TDD-driven execution in git worktrees:
- `brainstorming` — refine ideas into approved designs (or skip this if you already ran `/analyze-spec`)
- `writing-plans` — decompose into granular 2-5 min tasks with exact code
- `subagent-driven-development` — execute tasks via independent subagents
- `test-driven-development` — RED-GREEN-REFACTOR cycle enforcement

### Handoff Between Systems

After running `/analyze-spec`, use `/implement` to create an implementation brief, generate the execution plan via superpowers' `writing-plans`, and start execution.

```
Design Spec → /analyze-spec → Analysis Report → /implement → Implementation Brief → writing-plans → Execution Plan → Execute
```

## Artifacts

| Artifact | What It Is | Where It Lives |
|----------|-----------|----------------|
| Design Spec | User's input describing what to build | `specs/` |
| Analysis Report | Orchestrator's synthesized expert recommendations | `output/analysis-reports/` |
| Implementation Brief | Prioritized routing document referencing the analysis report | `output/briefs/` |
| Execution Plan | Granular task-based plan with exact code | `output/plans/` (generated by `writing-plans`) |

## Project Structure

```
.claude/
├── commands/                      # Slash commands
│   ├── analyze-spec.md            # Analyze design spec with domain experts
│   ├── implement.md               # Create brief, generate plan, offer execution
│   ├── eval-agent.md              # Evaluate agent output quality
│   └── review-cycle.md            # Post-implementation review and agent improvement
├── agents/
│   ├── orchestrator/              # Orchestrator agent
│   ├── domain-experts/            # 7 domain expert agents
│   ├── templates/                 # Agent template + shared prompt templates
│   └── tools/                     # Registry helper
config/
└── agents.yaml                    # Agent registry and configuration
specs/                             # User-written design specs (input)
├── template.md
└── examples/
output/                            # All generated content (gitignored)
├── analysis-reports/              # Saved analysis reports (from /analyze-spec)
├── eval-results/                  # Saved agent eval outputs (from /eval-agent)
├── review-cycles/                 # Post-implementation reviews (from /review-cycle)
├── briefs/                        # Implementation briefs (from /implement)
└── plans/                         # Execution plans (from writing-plans)
docs/
└── agentic-workflow/              # System documentation
```

## Key Conventions

- Design specs go in `specs/`
- All generated output goes in `output/` (gitignored)
- Analysis reports: `output/analysis-reports/`
- Implementation briefs: `output/briefs/`
- Execution plans: `output/plans/` (generated by `writing-plans`)
- Agent definitions are markdown files with YAML frontmatter
- The orchestrator reads `config/agents.yaml` to discover agents
- Adding a new agent requires no orchestrator changes — just add the file and register in `agents.yaml`
